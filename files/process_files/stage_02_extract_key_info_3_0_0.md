## 输出
### 用户画像
- 目标用户：应届生、1-3 年经验的技术岗求职者
- 典型方向：Java 后端、AI 应用开发
- 主要诉求：真实模拟、针对性追问、结构化反馈、持续复训

### 核心功能
1. 首页引导与快速开始
2. 创建面试
3. AI 多轮面试会话
4. 面试报告生成
5. 训练中心推荐
6. 历史记录回看
7. 个人中心与默认资料管理

### 核心数据对象
- User
- Resume
- KnowledgeAsset
- InterviewSession
- InterviewPlan
- QuestionNode
- AnswerRecord
- Report
- TrainingTask
- EventLog

### 核心业务流程
首页进入 -> 创建面试 -> 上传资料/补充信息 -> 生成面试计划 -> 进入面试房间 -> 多轮提问/追问 -> 结束面试 -> 生成报告 -> 进入训练中心或再次发起面试

### 页面流程
- 首页：首访转创建、回访转历史/训练
- 创建面试页：填写配置 -> 校验 -> 上传资料 -> 生成计划
- 面试房间：问题展示 -> 用户回答 -> 回答评估 -> 追问/切阶段 -> 结束
- 报告页：查看分数/建议 -> 再训练/查看历史
- 训练中心：按弱点推荐下一轮训练
- 历史记录：查看过往面试与报告
- 个人中心：管理默认简历、偏好和数据

### 关键交互要求
- 单题单答，追问紧贴上下文
- 即时反馈默认关闭
- 失败时保留用户输入内容
- 支持刷新后恢复会话
- 报告失败时可单独重试

### 非功能要求摘要
- 首页首屏 2 秒内可交互
- 问题生成 3-8 秒
- 创建面试 10 秒内给出结果或明确反馈
- 支持 Chrome、Edge、Safari 近两个大版本
- 支持移动端基本主流程

### 埋点重点
- 首页到创建页转化
- 创建成功率
- 面试完成率
- 报告查看率
- 再训练转化率

## 初步结论
- 产品主闭环明确，适合先围绕“创建 -> 会话 -> 报告”设计核心数据模型。
- 训练中心、历史记录和个人中心可以建立在同一套会话与报告数据之上。
