## 定稿说明
3.0.0 不再讨论“要不要做”，而是明确“页面长什么样、用户如何操作、失败时怎么处理、上线后如何观察效果”。

本版本重点回答以下问题：
- 每个页面的核心布局、组件和状态是什么。
- 每一条主流程在成功、失败、空状态下如何反馈。
- 面试过程中各种异常情况如何处理，避免中断体验。
- 埋点如何设计，确保上线后可以分析漏斗、留存和功能价值。

## 设计原则
- 开始成本低：让用户能在最短时间内开始第一场面试。
- 面试沉浸感优先：面试房间应避免信息干扰，聚焦问答本身。
- 反馈必须可执行：所有评分、建议、推荐训练都要说明依据。
- 复训路径清晰：从报告页到训练页的路径必须直接、明确。
- 状态可感知：加载、失败、成功、空状态都要给出清晰反馈。

## UI 视觉方向
### 风格关键词
- 专业
- 冷静
- 可信
- 聚焦

### 色彩建议
- 主色：深蓝或靛蓝，用于 CTA、重点信息和阶段标识。
- 辅色：青色或绿色，用于成功状态、积极反馈和推荐训练。
- 警示色：橙色，用于提醒用户信息不足、面试未完成。
- 错误色：红色，用于上传失败、网络错误、解析失败等。
- 背景：浅灰白为主，面试房间可使用更克制的浅色分区提升专注感。

### 字体与信息层级
- 标题：强调结构与节奏，建议 20-32px。
- 正文：保证长时间阅读舒适，建议 14-16px。
- 会话区文字：优先可读性，行高建议 1.6 左右。
- 数据卡片：数字和标签需明显分层，避免报告页信息拥挤。

## 页面原型与交互说明
### 1. 首页
#### 页面目标
- 让首次用户迅速理解产品价值。
- 让回访用户快速进入继续训练、查看报告或再次发起面试。

#### 桌面端布局建议
- 顶部导航：Logo、训练中心、历史记录、个人中心。
- Hero 区：价值主张、开始模拟面试 CTA、辅助说明。
- 中部内容区：推荐岗位方向、最近一次训练摘要、产品亮点。
- 底部信息区：常见问题、数据安全说明、版权信息。

#### 移动端布局建议
- 顶部简化为 Logo + 个人入口。
- 首屏只保留价值主张和主 CTA。
- 推荐模块改为纵向卡片流。

#### 初始状态
- 首次用户看到产品介绍、主按钮“开始模拟面试”。
- 老用户额外看到“继续上次训练”“查看最近报告”。

#### 触发操作
- 点击“开始模拟面试”进入创建面试页。
- 点击“训练中心”进入训练中心页。
- 点击“历史记录”查看过往报告。

#### 成功状态
- 正常跳转到目标页面。

#### 失败状态
- 页面配置加载失败时，展示错误提示和“重新加载”按钮。

#### 空状态
- 若用户暂无历史数据，则只展示推荐岗位和首次训练引导。

### 2. 创建面试页
#### 页面目标
- 以最低门槛完成必要配置。
- 用足够信息支撑个性化面试计划生成。

#### 桌面端布局建议
- 左侧：配置表单。
- 右侧：实时预览卡，包括目标岗位、面试类型、难度、资料状态、预估面试时长。

#### 页面字段
| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 岗位方向 | 单选 / 下拉 | 是 | Java 后端、AI 应用开发等 |
| 面试类型 | 单选 | 是 | 技术面、项目面、HR 面 |
| 难度 | 单选 | 是 | 初级、中级、高级 |
| 目标公司 | 输入 | 否 | 用于细化语境，不做强依赖 |
| 简历上传 | 文件上传 | 否但强引导 | 支持 PDF、DOCX、TXT |
| 补充资料 | 文本 / 文件 | 否 | 可补充项目背景、技能栈、JD |

#### 交互流程
1. 用户选择岗位、面试类型、难度。
2. 用户可上传简历或填写补充信息。
3. 系统实时校验必填项。
4. 点击“开始生成面试计划”后进入加载状态。
5. 生成成功后跳转到面试房间。

#### 初始状态
- 表单为空。
- 右侧显示默认预估时长和示例说明。

#### 成功状态
- 表单通过校验。
- 简历解析完成或资料提交成功。
- 进入面试房间。

#### 失败状态
- 上传失败：展示具体失败原因。
- 解析失败：允许跳过解析直接开始基础面试。
- 生成失败：提示重试，保留用户已填内容。

#### 空状态
- 未上传简历时，展示“上传简历将获得更贴合的问题”提示。

### 3. 面试房间
#### 页面目标
- 提供连续、沉浸、可信的 AI 面试体验。
- 明确阶段推进，弱化无关信息干扰。

#### 桌面端布局建议
- 顶部栏：当前岗位、面试类型、当前阶段、退出按钮。
- 左侧阶段导航：自我介绍、项目深挖、技术考察、场景追问、总结。
- 中间主区：AI 和用户的会话流。
- 右侧辅助区：资料摘要、本场设置、可选即时反馈开关。
- 底部输入区：回答输入框、发送、结束面试。

#### 移动端布局建议
- 阶段导航折叠为顶部步骤条。
- 辅助区收纳到抽屉或底部面板。
- 输入区固定在底部。

#### 关键组件
| 组件 | 作用 |
| --- | --- |
| 阶段步骤条 | 让用户知道当前面试进度 |
| 会话流 | 展示问题、回答和追问链路 |
| 输入框 | 用户主要操作区 |
| 状态提示 | 告知正在生成问题、正在生成追问等 |
| 结束确认弹窗 | 防止误触结束 |

#### 初始状态
- 展示开场说明。
- 高亮当前阶段“自我介绍”。
- AI 自动发出第一题。

#### 触发操作
- 用户输入并发送回答。
- 用户切换是否开启即时反馈。
- 用户主动结束面试。

#### 成功状态
- 回答发送成功。
- AI 在合理时延内返回下一题或追问。
- 阶段状态正常推进。

#### 失败状态
- 发送失败：输入内容保留并支持重发。
- 问题生成失败：提示重试或跳过到下一题。
- 会话超时：提示重新连接并恢复最近上下文。

#### 空状态
- 理论上不存在完全空状态。
- 若初始题生成失败，则展示“暂未生成问题，请重试”。

#### 特别交互要求
- 每次只展示一个主问题，追问应紧贴原问题。
- 即时反馈默认关闭，避免影响面试沉浸感。
- 用户若长时间未回复，可提示“是否需要更多时间”。

### 4. 面试报告页
#### 页面目标
- 让用户快速看懂本场表现。
- 让用户愿意继续训练，而不是只看分数后离开。

#### 桌面端布局建议
- 顶部摘要区：总体评价、总评分、主要标签。
- 中部左侧：维度评分、趋势摘要。
- 中部右侧：亮点与薄弱点。
- 下部：代表性问题复盘、建议、推荐训练主题。

#### 核心模块
| 模块 | 内容 |
| --- | --- |
| 总体评价 | 一句话总结本场表现 |
| 维度评分 | 技术能力、表达能力、项目深度、稳定性 |
| 问题复盘 | 代表性问题、用户回答摘要、问题定位 |
| 改进建议 | 可执行建议，不少于 3 条 |
| 推荐训练 | 下一轮训练主题、再次开始按钮 |

#### 初始状态
- 报告生成中显示骨架屏和进度说明。

#### 成功状态
- 展示完整报告。
- 提供“再次训练”“查看历史”“导出摘要”入口。

#### 失败状态
- 报告生成失败时支持重试。
- 若只生成部分内容，先展示已完成部分并提示稍后补全。

#### 空状态
- 若没有足够回答数据，展示“本场面试数据不足，建议重新完成一场完整面试”。

### 5. 训练中心
#### 页面目标
- 把单次报告转化为系统化复训。

#### 页面内容
- 弱点标签区。
- 推荐训练任务区。
- 最近训练完成情况。
- 再次发起训练按钮。

#### 状态说明
- 无训练任务时，展示“完成一次面试后将生成你的专属训练建议”。

### 6. 历史记录页
#### 页面目标
- 帮助用户快速找到过去的面试记录和报告。

#### 页面内容
- 记录列表。
- 筛选项：岗位、面试类型、时间。
- 快捷操作：查看报告、再次训练、删除记录。

#### 状态说明
- 无记录时展示空状态和首次训练 CTA。

### 7. 个人中心
#### 页面目标
- 管理用户资料和默认配置，降低重复输入成本。

#### 页面内容
- 基础信息。
- 默认岗位方向。
- 默认简历。
- 数据管理与删除入口。

## 通用交互规范
### 按钮规范
- 主按钮：用于开始面试、发送回答、再次训练。
- 次按钮：用于返回、取消、稍后再说。
- 危险按钮：用于删除记录、清空数据。

### 反馈规范
- 成功：Toast + 轻量状态变化。
- 警告：Toast 或行内提示，不阻断主流程。
- 错误：Toast + 明确说明 + 重试入口。
- 长任务：使用骨架屏、进度条或文案提示。

### 文案规范
- 避免“系统异常”这类无信息价值的提示。
- 优先告诉用户发生了什么、是否影响数据、下一步能做什么。

## 边缘情况处理
| 场景 | 处理策略 | 用户提示 |
| --- | --- | --- |
| 用户快速重复点击“开始面试” | 按钮进入 loading 并禁用重复提交 | 正在生成面试计划，请稍候 |
| 简历文件过大 | 前端限制大小并拒绝上传 | 文件过大，请上传 10MB 以内文件 |
| 简历格式不支持 | 阻止继续上传 | 当前仅支持 PDF、DOCX、TXT |
| 简历解析失败 | 允许跳过解析，走基础面试流程 | 简历解析失败，可继续开始通用面试 |
| 网络请求失败 | 保留用户已填内容并允许重试 | 网络异常，请稍后重试 |
| 生成问题超时 | 给出重试和跳过选项 | 当前问题生成较慢，请重试或跳到下一题 |
| 用户发送回答后接口失败 | 输入框保留原内容，支持重发 | 回答发送失败，请重试 |
| 用户刷新面试房间 | 自动恢复最近会话状态 | 已为你恢复最近的面试进度 |
| 用户误点结束面试 | 二次确认弹窗 | 确定结束本场面试吗 |
| 报告生成失败 | 支持单独重试报告生成 | 报告生成失败，请重新生成 |
| 历史记录为空 | 展示空状态和首次训练引导 | 还没有历史记录，先开始第一场面试吧 |
| 训练中心无推荐任务 | 引导先完成完整面试 | 完成一次面试后可获得专属训练建议 |
| 用户长时间未操作 | 弹出轻提示，不强制中断 | 需要更多时间准备回答吗 |
| 资料上传成功但生成失败 | 保留上传结果，不要求重复上传 | 面试计划生成失败，请重试创建 |
| 服务端返回内容不完整 | 展示已生成部分并标记未完成 | 当前内容未完全生成，可稍后刷新查看 |

## 页面级埋点方案
### 埋点目标
- 观察用户从首页到完成面试的漏斗转化。
- 判断简历上传、即时反馈、推荐训练等功能是否有价值。
- 分析用户流失点，优化创建面试和报告查看体验。

### 事件命名规范
- 使用英文小写 + 下划线命名。
- 统一包含页面名、事件动作、关键参数。
- 埋点字段尽量稳定，避免频繁变更。

### 推荐分析指标
- 首页到创建页转化率。
- 创建页到面试房间转化率。
- 面试开始到面试完成转化率。
- 报告页查看完成率。
- 从报告页进入再次训练的转化率。
- 上传简历用户与未上传简历用户的完成率差异。
- 即时反馈开启用户与关闭用户的完成率差异。

## 非功能性需求
| 类型 | 要求 | 说明 |
| --- | --- | --- |
| 性能 | 首页首屏建议 2 秒内可交互 | 降低首次流失 |
| 性能 | 创建面试提交后 10 秒内给出结果或明确反馈 | 避免等待焦虑 |
| 性能 | 单轮问题生成建议 3-8 秒 | 保证会话连续性 |
| 可用性 | 核心流程在 1366 宽桌面端体验稳定 | 兼顾主流办公设备 |
| 兼容性 | 支持 Chrome、Edge、Safari 最新两个大版本 | 避免浏览器兼容问题 |
| 响应式 | 支持移动端浏览与基本操作 | 至少保证可完成主流程 |
| 安全性 | 文件上传需校验格式、大小和安全性 | 防止恶意文件 |
| 隐私 | 支持用户删除简历、报告和历史记录 | 满足数据治理要求 |
| 可靠性 | 接口异常时不得丢失用户已输入内容 | 尤其是长回答文本 |
| 可观测性 | 前后端均应记录错误日志和关键链路耗时 | 便于排查线上问题 |
| 可访问性 | 颜色对比、按钮可点击区域、键盘输入需合格 | 满足基础无障碍要求 |

## 开发验收标准
### 首页
- 用户进入首页后 3 秒内能识别主 CTA。
- 新用户和老用户看到的首页内容有所区分。

### 创建面试页
- 必填项未填写时不能提交。
- 上传失败、解析失败、生成失败均有明确反馈。
- 提交失败后用户输入内容不丢失。

### 面试房间
- 首题可自动触发。
- 至少支持 10 轮连续对话。
- 追问与用户上一轮回答相关，不得明显脱节。
- 用户刷新页面后可恢复最近进度。

### 报告页
- 报告包含总评、维度评分、问题复盘、改进建议、训练推荐。
- 报告生成失败时可单独重试。

### 训练中心与历史记录
- 历史记录可查看最近报告。
- 报告页可直接进入再次训练。
- 无历史记录和无训练任务时均有清晰空状态。

## 测试关注点
- 首次使用路径是否顺滑。
- 长文本回答提交是否稳定。
- 网络抖动下是否会丢失用户输入。
- 上传简历后与未上传简历的面试体验差异是否符合预期。
- 即时反馈开关是否会影响主要问答链路。
- 页面在桌面端和移动端是否都能完成主流程。

## 研发实施建议
- 第一优先级先做首页、创建页、面试房间、报告页主链路。
- 第二优先级补历史记录、训练中心和个人中心。
- 埋点要和页面开发同步实施，不建议后补。
- 所有失败场景至少要给出“发生了什么”和“下一步怎么做”。
